<objective>
Add a programmatic screenshot helper `captureDashboard.m` to the Dashboard library so Claude (via matlab-mcp) can render a DashboardEngine, save it to a PNG, and Read the PNG. Supports full-dashboard capture, single-widget capture by handle or title, and detached-widget capture by following the DetachedMirror's standalone figure. Pure MATLAB, Octave-compatible, no API changes to DashboardEngine or DashboardWidget.

Purpose: Enables agentic UI testing of the Dashboard engine — Claude can build a dashboard, snapshot it, and visually inspect via the Read tool. This is the foundational primitive for option (B) screenshot-based UI verification.

Output:
- libs/Dashboard/captureDashboard.m (helper function)
- tests/test_capture_dashboard.m (function-style Octave-compatible test)
- examples/03-dashboard/example_capture_dashboard.m (matlab-mcp workflow demo)
</objective>

               Else (embedded):
                   require target.hFigure valid; hFig = target.hFigure.
                   On MATLAB only: targetObj = w.hPanel  (exportgraphics accepts a uipanel).
                   On Octave: targetObj = hFig (whole figure — print() doesn't reliably target uipanels). Document this limitation in the function header.

       - Else (Octave): stub-axes fallback. If targetObj is the figure, mirror existing logic from DashboardEngine.exportImage lines 462-475 (insert hidden 1px stub axes if no top-level axes, then `print(hFig, '-dpng', sprintf('-r%d', opts.Resolution), absPath)`, then delete stub).
       - Wrap in try/catch -> error captureDashboard:writeFailed with %s of ME.message.
    7. Restore BackgroundColor via onCleanup.
    8. Return absPath.
