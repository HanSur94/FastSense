<objective>
Fix the orange "no data" / stale-data banner overlapping the multi-page tab strip in `DashboardEngine`. Today both panels share the same vertical slot directly beneath the toolbar (`y = 1 - toolbarH - barH`), and the banner is `uistack`'d on top — so whenever it shows on a multi-page dashboard (e.g. the 6-page industrial demo) the page tabs become unclickable and invisible.

Reposition the banner BELOW the page-tab strip so chrome stays fully visible: `y = 1 - effToolbarH - effPageBarH - bannerH`. Single-page dashboards (where `effPageBarH == 0`) keep current behavior. Layout.ContentArea is NOT shrunk — banner remains a passive overlay covering the topmost sliver of widget content only when it shows.

Purpose: Restore tab navigation while a stale-data warning is up; eliminate the visual occlusion bug.
Output: Updated `DashboardEngine.m` (new `repositionStaleBanner_` helper + adjusted createStaleBanner offset + invocations from `applyVisibilityAndRelayout` and `onResize`); one new regression test case in `tests/test_dashboard_stale_banner.m`.
</objective>
