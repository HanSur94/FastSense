# Plan: Dashboard Info button opens modal in-app window

## Objective

When a user clicks the toolbar Info button in interactive MATLAB desktop sessions, render the Markdown-derived HTML inside a modal MATLAB `uifigure` with a `uihtml` panel, instead of shelling out to an external browser via `web()`. Keep the headless/Octave codepath (`system('open ...')` etc.) exactly as is.

## Context

- `DashboardEngine.writeAndOpenInfoHtml` currently writes HTML to a temp file, then on MATLAB desktop calls `web(obj.InfoTempFile, '-new')`.
- Existing test suite (`TestDashboardInfo.m`) relies on `InfoTempFile` being populated and `cleanupInfoTempFile()` deleting it.
- New behaviour: in interactive MATLAB desktop, build a modal `uifigure` with a `uihtml` control whose `HTMLSource` is the rendered HTML. The temp file must still be written for backward compatibility and for tests.

## Tasks

### Task 1 — Add `showInfoModal_` codepath in DashboardEngine

type=auto

- Add a private property `InfoModalFigure_ = []` to track the modal figure handle (so repeated clicks reuse / refocus rather than stacking).
- Add private method `showInfoModal_(obj, html)` that:
  - If `InfoModalFigure_` is a valid handle, brings it to front and updates its `HTMLSource` instead of opening a new figure.
  - Else creates `uifigure('Name', ['Info — ' obj.Name], 'WindowStyle', 'modal', 'Visible', 'on')` with a sensible default size (e.g. 800x600 centered).
  - Adds a `uihtml` filling the figure and sets `HTMLSource = html`.
  - Stores handle in `InfoModalFigure_`; `CloseRequestFcn` clears the property and `delete`s the figure.
- Modify `writeAndOpenInfoHtml(obj, html)`:
  - Always write the temp HTML file (unchanged — preserves test contract).
  - Octave branch: unchanged (`system('open ...')` etc.).
  - MATLAB branch: replace `web(obj.InfoTempFile, '-new')` with a guarded modal call. Guard:
    `~exist('OCTAVE_VERSION','builtin') && usejava('desktop') && ~(exist('batchStartupOptionUsed','builtin') && batchStartupOptionUsed())`.
  - Inside the guard, call `obj.showInfoModal_(html)`.
  - Outside the guard (headless MATLAB), do nothing extra — temp file is still written.
- Update `cleanupInfoTempFile` (or add cleanup in `delete`) so the modal figure is also closed if open.

Verification:
- `mcp__matlab__check_matlab_code` on `libs/Dashboard/DashboardEngine.m` returns no new lint errors.
- Grep confirms `uihtml` symbol present and `web(obj.InfoTempFile` no longer present.

### Task 2 — Update TestDashboardInfo

type=auto

- Add a new test `testShowInfoOpensModalFigure` that:
  - Skips if `~usejava('desktop')` (headless CI) — `testCase.assumeTrue(usejava('desktop'))`.
  - Calls `d.showInfo()` then verifies `d.InfoModalFigure_` is a valid handle and a `uihtml` child exists.
  - Adds teardown to close the modal and call `cleanupInfoTempFile`.
- Add `testShowInfoModalReusesFigure` that:
  - Skips on headless.
  - Calls `showInfo()` twice, verifies the same figure handle is reused (not stacked).
- Existing tests must remain green — we keep `InfoTempFile` populated. No edits to existing tests except ensure that adding teardown for modal handle does not regress.

Verification:
- `mcp__matlab__check_matlab_code` on `tests/suite/TestDashboardInfo.m` shows no new errors.
- Grep confirms the two new test method names appear.

- `InfoTempFile` is still populated (backward compatibility).
- `cleanupInfoTempFile` still works.
