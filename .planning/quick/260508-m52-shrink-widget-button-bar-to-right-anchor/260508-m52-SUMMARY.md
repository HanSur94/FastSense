# Quick 260508-m52: Shrink WidgetButtonBar to 64px Right-Anchored Strip

One-liner: Restore the documented "small opaque strip in the top-right" behavior of the per-widget button bar so widget titles (GroupWidget header, StatusWidget label, TextWidget title, BarChartWidget axes title, DividerWidget body) are no longer obscured.

## Problem

`DashboardLayout.getOrCreateButtonBar_` had drifted from its doc-comment ("small opaque strip in the top-right") to render the bar at full panel width:

```matlab
barW = pp(3) - 2 * inset;   % full width
x    = inset;                % left-anchored
```

Combined with `barH = 28` and `BackgroundColor = theme.GroupHeaderBg`, this drew an opaque header band that covered each widget's own title rendering. Visually obvious in the industrial demo's "Feed Line" tab (Feed Line Signals group header, Feedline Pressure High status title, About Feed Line text title, BarChart axes title all clipped).

`reflowButtonBar_` had the same bug, so the cover-up survived panel resizes too.

## Fix

Two-line formula change in both `getOrCreateButtonBar_` (~L600-604) and `reflowButtonBar_` (~L697-699):

```matlab
barW = min(64, max(1, pp(3) - 2 * inset));
x    = max(inset, pp(3) - barW - inset);
```

64px is sized to host the existing info(24px) + spacing(4px+4px) + detach(24px) + edge-padding(8px) = ~64px. Buttons reflow within `barW` using their existing `barW - 24 - 4` / `barW - 24 - 24 - 4 - 4` formulas — no per-button changes needed.

Doc-comment at `getOrCreateButtonBar_` updated to match new behavior.

## Test Seam

`reflowButtonBar_` was `Access = private` Static — moved to a new `methods (Static)` (public) block so the regression test can drive a deterministic resize. The runtime call site (the SizeChangedFcn lambda inside `getOrCreateButtonBar_`) is unaffected. `anchorTopRight` stays in its private Static block.
