# Phase 1027.1 Plan 07: update-companion-tests Summary

Migrated `tests/suite/TestFastSenseCompanion.m` from Phase 1027 single-pane log accessors to Phase 1027.1 paired/parameterized accessors, and added 5 independence tests verifying that detaching, hiding, or closing the events log does not affect the live log (and symmetric for live).

## What Shipped

**File modified:** `tests/suite/TestFastSenseCompanion.m`

### Accessor migrations (13 edits across 11 existing tests)

| Old (Phase 1027) | New (Phase 1027.1) | Sites |
|---|---|---|
| `app.getLogPane()` | `app.getEventsLogPane()` or `app.getLiveLogPane()` (per intent) | 4 |
| `app.getLogStateValue()` | `app.getEventsLogStateValue()` | 6 |
| `app.getDetachedLogFig()` | `app.getDetachedEventsFig()` | 12 |
| `app.applyLogState('X')` | `app.applyLogState('events', 'X')` | 18 |
| `pane.bufferSize('log')` | `pane.bufferSize()` (single buffer per pane) | 2 |
| `pane.bufferSize('live')` | `pane.bufferSize()` | 1 |

`getRow3Height()` and `getLiveButtonParent()` were unchanged (they remained single-valued in Plan 05).

### Tests renamed

- `testLogStateDefaultIsInline` → `testEventsAndLiveLogStateDefaultIsInline` (now asserts both panes' Inline + IsAttached + row-3 height)
- `testLogStateInlineToDetachedToInline` → `testEventsLogStateInlineToDetachedToInline` (events-targeted; row-3 stays at 360 because live is still Inline)
- `testLogStateInlineToHiddenToInline` → `testEventsLogStateInlineToHiddenToInline` (row-3 stays at 360 — semantic change from Phase 1027)
- `testLogEntriesSurviveHiddenState` → `testEventsLogEntriesSurviveHiddenState`
- `testProgrammaticCloseDrivesStateToInline` → `testEventsProgrammaticCloseDrivesStateToInline`

`testThemeSwitchWhileDetached`, `testThemeSwitchWhileHidden`, `testSetLogStateIdempotent`, `testCloseWhileDetachedDoesNotRecurse` kept their names but had their bodies migrated to the events-targeted parameterized API (single-pane intent preserved).

`testLiveButtonParentedToToolbar` and `testLiveButtonStillTogglesLiveMode` were untouched — they don't use any log accessors.

### New independence tests added (Phase 1027.1)

A new `% --- Phase 1027.1 independence tests ---` section adds:

1. **`testEventsDetachedDoesNotAffectLive`** — detach events → live stays Inline + IsAttached + row 3 stays at 360.
2. **`testLiveDetachedDoesNotAffectEvents`** — detach live → events stays Inline + IsAttached + row 3 stays at 360. Detached live uifigure size verified at 480×360.
3. **`testEventsHiddenLiveInlineFillsStrip`** — Hide events → outer row 3 still 360, inner sub-grid `hLogStripGrid_.RowHeight = {0, '1x'}` (live fills the strip). Uses `struct(app)` with the standard `MATLAB:structOnObject` warning suppression.
4. **`testBothDetachedCollapsesOuterRow`** — both detached → outer row 3 collapses to 0; both detached uifigures exist.
5. **`testCloseEventsDetachedWindowOnlyAffectsEvents`** — both detached, then `figE.CloseRequestFcn(figE,[])` → events back to Inline, live still Detached + uifigure intact (validates Plan 05's `CloseRequestFcn = @(~,~) setLogState_('events','Inline')` per-pane scoping).
