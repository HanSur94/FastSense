# Phase 1027.1 Plan 02: Create LiveLogPane Summary

Pure port of the live-half of `LogPane` into a new self-contained `LiveLogPane` handle class, plus a brand-new pop-out icon button so the live updates pane can be detached independently of the events log.

## Public API Surface

| Method | Purpose |
| ------ | ------- |
| `LiveLogPane(themeStruct)` | Constructor; validates theme struct, initializes empty buffer. UI is NOT built. |
| `attach(parent, themeStruct)` | Builds the 2-row live-updates layout inside parent (uipanel or uifigure). Idempotent. |
| `detach()` | Destroys hRoot_ subtree; preserves LiveLogBuffer_. |
| `addLiveLogEntry(tagKey, deltaSamples, latestY)` | Append to live-updates log. Buffers detached, renders attached. 500-row cap. |
| `clearLiveLog()` | Wipe LiveLogBuffer_ and the live table. |
| `applyTheme(themeStruct)` | Live theme switch via applyThemeToChildren_ walker + accent re-assertion. |
| `delete()` | Destructor wraps detach() in try/catch. |
| `bufferSize()` | Test helper: row count of LiveLogBuffer_ (no `which` arg — single buffer in this pane). |
| `peekLiveRow(idx)` | Test helper: read row idx from LiveLogBuffer_ (newest-first). |
| `rootBackgroundColor()` | Test helper: read hRoot_.BackgroundColor (or [] when detached). |
| `requestDetach()` | Programmatic equivalent of clicking the pop-out icon. |

Event: `DetachRequested` (no payload) — fired when user clicks the inline pop-out icon button or `requestDetach()` is called.

## What Was Ported Verbatim from LogPane.m

- Live header label + Clear button cells (LogPane.m lines 174-184)
- Live table column spec: `{'Time', 'Tag', char([8710, ' samples']), 'Latest'}` with widths `{65, 'auto', 90, 90}`, FontName 'Menlo', FontSize 10 (LogPane.m lines 187-197)

- Stripe-pair logic: `mean(t.DashboardBackground) < 0.5` -> `[0.13 0.13 0.13; 0.20 0.20 0.20]` for dark, `[1.00 1.00 1.00; 0.94 0.94 0.94]` for light

## New Pop-Out Icon (vs Phase 1027 LogPane)

Phase 1027's `LogPane` had **a single pop-out icon** on the events header that detached *both* logs as one unit. The original LogPane live header was a 1x2 grid (`{'1x', 80}`: label / Clear). It had **no pop-out** of its own.

Phase 1027.1 extends this header to **1x3** (`{'1x', 80, 36}`: label / Clear / pop-out) so each log strip can detach independently. The new pop-out:

- Layout: `Layout.Row=1, Layout.Column=3`
- Glyph: `char(8689)` (pop-out arrow, CLAUDE.md compliant)
- FontSize: 14
- Tooltip: `'Detach live log to its own window'` (note: "live log", not just "log" — disambiguates from EventsLogPane's twin)
- Background: `t.WidgetBorderColor`; FontColor: `t.ForegroundColor` — matches the settings-gear styling convention from FastSenseCompanion
- Callback: `@(~,~) notify(obj, 'DetachRequested')` — fires the public event

## LiveSampleCount_ Boundary

The class **does not declare** `LiveSampleCount_` (or any per-tag last-seen-count state). This is a deliberate Phase 1027 boundary preserved into 1027.1: pipeline-cursor state lives on `FastSenseCompanion`, not on the pane. The pane is a passive renderer that receives pre-computed `deltaSamples` from the companion. Verified by `grep -c "LiveSampleCount_"` → 0.

## applyTheme Order (Phase 1027 Fix Lesson)

`applyTheme` sets `obj.hRoot_.BackgroundColor = t.WidgetBackground` **before** calling `applyThemeToChildren_(obj.hRoot_, themeStruct)`. The walker does not restyle the root layout itself — only its descendants — so the root must be coloured up-front. After the walker, the pop-out button colors and table stripe pair are re-asserted because the generic walker overwrites them with default ForegroundColor styling.
