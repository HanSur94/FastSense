<migration_pattern>
<!-- The exact text-substitution pattern for every Threshold(...) ref. -->
<!-- This is a PURE SHIM migration: caller-side semantics unchanged, construction call site swapped. -->

Legacy code (3 lines per ref):
```matlab
t = Threshold('warn', 'Name', 'warn', 'Direction', 'upper');
t.addCondition(struct(), 10);
s.addThreshold(t);                            % `s` is a SensorTag
```

Becomes (1 line):
```matlab
m = MakeV21Fixtures.makeThresholdMonitor('warn', s, 10, 'upper');
```

Notes:
- The `'Name'` NV pair is dropped — MonitorTag uses Key for identification; no test asserts on the 'Name' string of a Threshold, so this is safe.
- `addThreshold` on the parent SensorTag is replaced by the MonitorTag's parent-tag wiring inside the constructor. If the test downstream calls `s.Thresholds` to enumerate threshold objects, those assertions need updating (rare — most tests just trigger detection and check event count).
- The TagRegistry is cleared in EVERY migrated test's setup (most already do this; if not, add `TagRegistry.clear()` at the top of the function or its `setup()`).
</migration_pattern>
