# Phase 1020 — UI Design Contract

> Visual and interaction contract for the Dashboard Browser pane.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Scope: MIDDLE PANE ONLY. Name search field, per-row Open button, status dot,
> widget count, empty/no-match state. All spacing/typography/color tokens
> inherit from Phase 1018's contract — this spec references them by name rather
> than redefining them. New declarations for Phase 1020 are clearly marked.
>
> Out of scope for this phase: inspector content for the dashboard-selected state
> (Phase 1021), live-updating dot reflecting `IsLive` changes in real time
> (BROWSER-06 deferred), Global "Pause All" toolbar (BROWSER-07 deferred).

| Property | Value |

## Color

All color values inherited from Phase 1018. See `1018-UI-SPEC.md §Color`.
Phase 1020 applies the inherited palette as follows:

### Dark preset (default)

| Role | RGB (inherited from Phase 1018) | Phase 1020 elements |
|------|-------------------------------|---------------------|
| Dominant (60%) — figure bg | [0.10, 0.10, 0.18] | Not directly used in pane interior |
| Secondary (30%) — pane bg | [0.09, 0.13, 0.24] | Middle-pane `uipanel.BackgroundColor`; `uieditfield.BackgroundColor`; all `uigridlayout.BackgroundColor` within pane |
| Accent (10%) | [0.31, 0.80, 0.64] | Live status dot `uilabel.FontColor`; "Open" button border color on hover (see Row States below) |
| Pane border | [0.16, 0.23, 0.37] | `uipanel.BorderColor`; "Open" button `BackgroundColor` (idle — same as pane border provides subtle outline effect) |
| Body / muted text | [0.66, 0.73, 0.78] | Idle status dot `FontColor`; widget-count label `FontColor`; count badge `FontColor`; search placeholder; clear button |
| Foreground | `theme.ForegroundColor` | Dashboard name text in row button; search field typed text; "Open" button text |

### Row background states

| State | `BackgroundColor` | Notes |
|-------|------------------|-------|
| Idle | `theme.WidgetBackground` | Same as pane background — rows blend into the surface |
| Selected | `theme.WidgetBorderColor` (= [0.16, 0.23, 0.37]) | Subtle highlight using the pane border color; no heavy contrast inversion |
| Focused (keyboard) | Same as Selected — MATLAB `uibutton` receives native focus ring; no additional overlay needed | |

**Selected-row rendering approach:** When the user clicks a row area button
(`DashboardSelected` fires), the pane iterates `hRowButtons_` and sets:
- Clicked row: `btn.BackgroundColor = theme.WidgetBorderColor`
- All other rows: `btn.BackgroundColor = theme.WidgetBackground`

This gives a single-selection highlight. The Open button and status dot within
the selected row are not recolored — only the row background changes.

### Accent reserved for (Phase 1020)

Accent (`theme.Accent`) is used on exactly these elements — no others:

1. Live status dot `uilabel.FontColor` when `DashboardEngine.IsLive == true`
2. "Open" button `BackgroundColor` is NOT accent — the button uses a contained
   style (see Component Inventory). Accent on the Open button would compete with
   the status dot meaning (live = accent).

Accent is NOT used on: row backgrounds, count badge, search field, selected-row
highlight, widget-count text, pane borders, or the Open button background.

### Destructive

No destructive actions in Phase 1020. `removeDashboard(key)` is a programmatic
API — no UI button, no confirmation dialog needed in this phase.
`theme.StatusAlarmColor` = [0.91, 0.27, 0.38] is not used in this phase.

**Source:** Phase 1018 UI-SPEC `§Color`, 1020-CONTEXT.md decisions ("Status dot
rendering: `theme.Accent` (live) or `theme.PlaceholderTextColor` (idle)").

## Visual Focal Point Declaration

**Required by Phase 1018 UI-SPEC review flag:** Phases 1019–1021 must declare
their focal point explicitly.

Phase 1020 focal mass: **dashboard row list** (scrollable column of dashboard
rows, occupying ~80% of the pane height). This is where the engineer's eye lands
first — the list of projects they can open.

The search field is the entry affordance above the list — it narrows the focal
mass but is not itself the destination. The count badge below is tertiary
informational feedback.

Layout vertical order (top to bottom within the middle-pane `uipanel`):

1. Search row (search field + clear button) — 28 px height
2. 8 px gap
3. Dashboard row list area — flexible (takes remaining height); rows stacked top-to-bottom
4. 4 px gap
5. Count badge `uilabel` — 24 px height

The middle pane `uipanel` inner content uses a `uigridlayout` with:

```
uigridlayout(hMidPanel_, [5 1])
  RowHeight   = {28, 8, '1x', 4, 24}
  ColumnWidth = {'1x'}
  Padding     = [16 16 16 16]   % theme.PanePadding all sides
  RowSpacing  = 0               % spacing baked into fixed rows
  BackgroundColor = theme.WidgetBackground
```

Row 3 (`'1x'`) contains a scrollable inner `uigridlayout` (or stacked row
panels) that holds the individual dashboard rows. See "Row Scroll Container"
below for the scroll approach.

The search row uses a nested 1×2 grid (mirrors Phase 1019 search row):

```
uigridlayout(searchRow, [1 2])
  ColumnWidth   = {'1x', 24}    % search field flexible; × button fixed 24 px
  RowHeight     = {'1x'}
  Padding       = [0 0 0 0]
  ColumnSpacing = 4
```

**Source:** 1020-CONTEXT.md decisions, Phase 1018 focal-point requirement,
Phase 1019 UI-SPEC (search row pattern mirrored).

## Row Anatomy

Each dashboard row is a `uigridlayout` parented to the scroll container, with
fixed height of 32 px. The row grid has 4 columns:

```
uigridlayout(scrollContainer, [1 4])
  ColumnWidth   = {'1x', 'fit', 16, 52}
  %               name  count  dot  open-btn
  RowHeight     = {32}
  Padding       = [0 2 0 8]     % top=0, right=2, bottom=0, left=8 px inner inset
  ColumnSpacing = 4
  BackgroundColor = theme.WidgetBackground
```

Column assignments:

| Col | Width | Element | Primitive |
|-----|-------|---------|-----------|
| 1 | `'1x'` | Dashboard name (clickable row area) | `uibutton` styled flat |
| 2 | `'fit'` | Widget count `(N widgets)` | `uilabel` |
| 3 | 16 px | Status dot `●` | `uilabel` |
| 4 | 52 px | "Open" button | `uibutton` |

**Column 1 — Row area button:**

The entire column 1 area is a `uibutton` styled flat (no visible border, no
background shadow). It acts as the row-selection affordance.

```matlab
btn.Text             = engine.Name        % DashboardEngine.Name
btn.FontSize         = 11
btn.FontWeight       = 'bold'
btn.FontColor        = theme.ForegroundColor
btn.BackgroundColor  = theme.WidgetBackground
btn.HorizontalAlignment = 'left'
btn.ButtonPushedFcn  = @(~,~) obj.onRowClicked_(idx)   % fires DashboardSelected
btn.Tooltip          = engine.Name        % full name on hover (truncation safety)
```

**Column 2 — Widget count label:**

```matlab
lbl.Text             = sprintf('(%d)', widgetCount)
lbl.FontSize         = 11
lbl.FontWeight       = 'normal'
lbl.FontColor        = theme.PlaceholderTextColor
lbl.HorizontalAlignment = 'left'
```

Widget count formula (from 1020-CONTEXT.md):
```matlab
widgetCount = numel(d.Widgets) + sum(cellfun(@(p) numel(p.Widgets), d.Pages));
```

**Column 3 — Status dot:**

```matlab
lbl.Text      = char(9679)   % Unicode ● U+25CF
lbl.FontSize  = 11
if d.IsLive
    lbl.FontColor = theme.Accent
else
    lbl.FontColor = theme.PlaceholderTextColor
end
lbl.HorizontalAlignment = 'center'
lbl.Tooltip   = 'Live'   % or 'Idle' — changes with IsLive state
```

`char(9679)` is the portable MATLAB way to embed `●` without file encoding issues.

**Column 4 — Open button:**

```matlab
btn.Text             = 'Open'
btn.FontSize         = 11
btn.FontWeight       = 'normal'
btn.FontColor        = theme.ForegroundColor
btn.BackgroundColor  = theme.WidgetBorderColor   % subtle contained appearance
btn.ButtonPushedFcn  = @(~,~) obj.onOpenClicked_(idx)   % fires OpenDashboardRequested
btn.Tooltip          = sprintf('Open "%s" in its own figure', engine.Name)
```

**Source:** 1020-CONTEXT.md decisions ("Row primitive: `uigridlayout` of custom
row containers", "Status dot rendering: Unicode `●` glyph in a `uilabel`",
"Row-click vs Open-click discrimination: wider row button fires `DashboardSelected`,
the Open button fires `OpenDashboardRequested`").

## Scroll Container

MATLAB `uigridlayout` does not natively scroll. The row list container is a
`uipanel` with `Scrollable = 'on'` (MATLAB R2020b+) parented to row-3 of the
outer pane grid. Inside it, a `uigridlayout` grows as rows are added:

```matlab
hScroll = uipanel(hOuterGrid, ...
    'Scrollable',      'on', ...
    'BorderType',      'none', ...
    'BackgroundColor', theme.WidgetBackground);
hScroll.Layout.Row    = 3;
hScroll.Layout.Column = 1;

hRowGrid = uigridlayout(hScroll, [nRows 1]);
hRowGrid.RowHeight     = repmat({32}, 1, nRows);
hRowGrid.ColumnWidth   = {'1x'};
hRowGrid.Padding       = [0 0 0 0];
hRowGrid.RowSpacing    = 2;   % 2 px gap between rows
hRowGrid.BackgroundColor = theme.WidgetBackground;
```

Each row container is then parented into `hRowGrid` at its assigned grid row.

When the list is rebuilt (`refresh()`), all children of `hRowGrid` are deleted
and new row containers are created. The scroll position resets to top on rebuild.
Handle: `hScroll_`, `hRowGrid_`.

**Source:** MATLAB `uipanel('Scrollable','on')` is the standard approach for
scrollable lists in programmatic uifigure (R2020b+). Row height 32 px × N rows
gives total inner height; `uipanel` provides the viewport and scrollbar.

## Interaction States

### State 1: Idle (dashboards loaded, no search, nothing selected)

- Search field: empty, placeholder `'Search dashboards…'` visible
- Row list: all dashboards shown, in input order (no sorting)
- All row backgrounds: `theme.WidgetBackground`
- Status dots: colored per `IsLive` snapshot at last `refresh()` (no live update)
- Count badge: `'{N} of {N} visible'` (N = total dashboard count)
- `SelectedDashboardIdx`: not set (orchestrator concern; pane holds no selection state)

### State 2: Empty (no dashboards loaded or all removed)

- Scroll container: shows `'No dashboards loaded'` centered label
- Count badge: `'0 of 0 visible'`
- Search field: still active (does not disable)

### State 3: Typing (debounce in progress)

- Search field: user has typed; text visible
- Debounce timer running (150 ms from last keystroke)
- Row list: unchanged until timer fires
- Count badge: unchanged

### State 4: Filtering (search active, results present)

- Row list rebuilt from `filterDashboards(engines, searchTerm)` output
- Only matching rows shown; row order matches input order within filtered set
- Count badge: `'{visible} of {total} visible'`
- Previously selected row highlight cleared (selection was engine-index based;
  rebuild resets highlight to show no selection — the orchestrator tracks
  `SelectedDashboardIdx` separately)

### State 5: No-match (filter active, zero results)

- Scroll container: shows `'No dashboards match'` centered label
- Count badge: `'0 of {total} visible'`

### State 6: Row selected (user clicked row area)

- Clicked row button: `BackgroundColor = theme.WidgetBorderColor`
- All other row buttons: `BackgroundColor = theme.WidgetBackground`
- `DashboardSelected` event fired with `DashboardEventData(engine, idx)`
- Orchestrator sets `LastInteraction = 'dashboard'`, `SelectedDashboardIdx = idx`

### State 7: Open clicked

- `OpenDashboardRequested` event fired (separate from `DashboardSelected`)
- Pane wraps `engine.render()` in try/catch:
  - Success: `engine.render()` creates its own classical `figure` — companion unchanged
  - Failure: `uialert(obj.hFig_, sprintf('Failed to open dashboard "%s": %s', engine.Name, ME.message), 'FastSense Companion')`
- Row selection highlight is also applied on Open click (clicking Open implies
  row focus — reuse `onRowClicked_` highlight logic before the open call)

### State 8: Dashboard added (`addDashboard(d)`)

- `FastSenseCompanion.addDashboard(d)` validates (non-duplicate, is `DashboardEngine`),
  appends to `Engines_`, calls `ListPane_.refresh()`
- `refresh()` rebuilds row list preserving current search string
- New dashboard row appears at the bottom of the unfiltered list
- Scroll position resets to top after rebuild

### State 9: Dashboard removed (`removeDashboard(key)`)

- Orchestrator removes engine from `Engines_` by `Name` match
- `ListPane_.refresh()` called; row for removed dashboard disappears
- If removed dashboard was selected: orchestrator clears `SelectedDashboardIdx`,
  inspector returns to placeholder/welcome (Phase 1021 concern — Phase 1020
  ensures the event is not re-fired for the deleted row)

### Debounce timer lifecycle

- Created lazily on first keystroke (not in `attach`) — zero timers when no search typed
- `ExecutionMode = 'singleShot'`, `Period = 0.150`
- `BusyMode = 'drop'`
- Each keystroke: `stop(timer); start(timer)` — resets countdown
- `TimerFcn`: calls `obj.applyFilter_()`
- Esc shortcut: calls `applyFilter_()` directly (synchronous, bypasses timer)
- On `detach()`: `stop(t); delete(t);` in that order (cross-cutting rule)
- Handle: `DebounceTimer_` private property; nil-checked before stop/start

**Source:** 1020-CONTEXT.md decisions ("Search debounce delay: 150 ms"),
STATE.md cross-cutting constraint ("stop(t); delete(t); always in that order").

## `attach` Signature

Phase 1020 extends the `DashboardListPane.attach` signature:

```matlab
function attach(obj, parentPanel, hFig, engines, theme)
```

- `parentPanel` — `uipanel` handle from `FastSenseCompanion.hMidPanel_`
- `hFig` — `uifigure` handle from `FastSenseCompanion.hFig_` — for `uialert`
- `engines` — cell array of `DashboardEngine` handles (snapshot from `Engines_`)
- `theme` — resolved `CompanionTheme.get(preset)` struct

`refresh()` receives no arguments — it reads `Engines_` via a reference captured
at `attach` time. The orchestrator calls `ListPane_.refresh()` after any
`addDashboard`/`removeDashboard` mutation. Alternatively, `refresh(engines)` may
accept an updated engines cell if the capture model is awkward — the executor
chooses based on what keeps cyclomatic complexity under 30.

`FastSenseCompanion` must update its `attach` call site to pass `hFig_` and
`Engines_` (mirrors Phase 1019 pattern: `TagCatalogPane.attach(panel, hFig, registry, theme)`).

**Source:** 1020-CONTEXT.md decisions ("Phase 1020 must update both call sites to
pass `hFig_` to `DashboardListPane.attach`"), Phase 1019 UI-SPEC `attach` signature.
